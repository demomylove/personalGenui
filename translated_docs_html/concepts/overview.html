核心概念 (Core Concepts)

本节解释 A2UI 的基本架构。理解这些概念将帮助您构建有效的代理驱动界面。

大图景 (The Big Picture)

A2UI 围绕三个核心思想构建：

  1. 流式消息 (Streaming Messages): UI 更新作为一系列 JSON 消息从代理流向客户端
  2. 声明式组件 (Declarative Components): UI 被描述为数据，而不是被编程为代码
  3. 数据绑定 (Data Binding): UI 结构与应用程序状态分离，从而实现响应式更新

关键主题

数据流 (Data Flow)

消息如何从代理传输到渲染的 UI。包括餐厅预订流程的完整生命周期示例、传输选项（SSE、WebSockets、A2A）、渐进式渲染和错误处理。

组件结构 (Component Structure)

A2UI 用于表示组件层级的 邻接表模型 (adjacency list model)。了解为什么扁平列表优于嵌套树，如何使用静态与动态子组件，以及增量更新的最佳实践。

数据绑定 (Data Binding)

组件如何通过 JSON Pointer 路径连接到应用程序状态。涵盖响应式组件、动态列表、输入绑定，以及使 A2UI 强大的结构与状态分离。

消息类型

A2UI 使用四种消息类型：

  • surfaceUpdate: 定义或更新 UI 组件
  • dataModelUpdate: 更新应用程序状态
  • beginRendering: 信号通知客户端渲染
  • deleteSurface: 移除 UI surface
