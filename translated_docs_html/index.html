代理驱动界面 (Agent-Driven Interfaces) 协议

A2UI 使 AI 代理能够生成丰富、交互式的用户界面，并在 Web、移动端和桌面上原生渲染——所有的这一切都无需执行任意代码。

⚠️ 状态: 早期公开预览版
A2UI 目前处于 v0.8 (Public Preview) 阶段。规范和实现已具备功能，但仍在不断演进。我们要开源该项目是为了促进协作、收集反馈并征集贡献（例如在客户端渲染器方面）。请预期会有变更。

概览 (At a Glance)

A2UI 目前版本为 v0.8，使用 Apache 2.0 许可证，由 Google 创建，CopilotKit 和开源社区共同贡献，并在 GitHub 上积极开发中。

A2UI 解决的问题是：AI 代理如何安全地跨越信任边界发送丰富的 UI？

A2UI 不使用纯文本响应或有风险的代码执行，而是让代理发送 声明式的组件描述，客户端使用其自己的原生小部件进行渲染。这就像拥有了一种代理通用的 UI 语言。

在这个仓库中，你可以找到： A2UI 规范，以及为了客户端准备的 渲染器（如 Angular, Flutter 等）实现，和在代理与与客户端之间传递消息的 传输层（如 A2A 等）实现。

🛡️ 安全设计 (Secure by Design)

声明式数据格式，而非可执行代码。代理只能使用您目录中预先批准的组件——没有 UI 注入攻击。

🚀 对 LLM 友好 (LLM-Friendly)

扁平、流式的 JSON 结构，专为易于生成而设计。LLM 可以增量构建 UI，而无需一次性生成完美的 JSON。

📱 框架无关 (Framework-Agnostic)

一个代理响应适用于任何地方。在 Angular、Flutter、React 或原生移动端上渲染相同的 UI，并使用您自己的样式化组件。

📈 渐进式渲染 (Progressive Rendering)

随着生成的进行流式传输 UI 更新。用户可以看到界面实时构建，而不是等待完整的响应。

工作原理 (How It Works)

  1. 用户发送消息 给 AI 代理
  2. 代理生成 A2UI 消息 描述 UI（结构 + 数据）
  3. 消息流式传输 到客户端应用程序
  4. 客户端渲染 使用原生组件（Angular, Flutter, React 等）
  5. 用户交互 UI，将操作发送回代理
  6. 代理响应 更新后的 A2UI 消息
