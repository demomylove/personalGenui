A2UI 用在哪里? (Where is A2UI Used?)

A2UI 正被 Google 和合作伙伴团队采用，以构建下一代代理驱动的应用程序。以下是 A2UI 正在产生影响的真实案例。

生产环境部署 (Production Deployments)

Google Opal: AI Mini-Apps for Everyone

Opal 使成千上万的人能够使用自然语言构建、编辑和共享 AI 迷你应用程序——无需编码。

Opal 如何使用 A2UI：

Google 的 Opal 团队从一开始就是 A2UI 的核心贡献者。他们使用 A2UI 来驱动使 Opal 的 AI 迷你应用程序成为可能的动态、生成式 UI 系统。

  • 快速原型设计: 快速构建和测试新的 UI 模式
  • 用户生成的应用程序: 任何人都可以创建具有自定义 UI 的应用程序
  • 动态界面: UI 自动适应每个用例

"A2UI 是我们工作的基础。它给了我们灵活性，让 AI 以新颖的方式驱动用户体验，而不受固定前端的限制。其声明式性质和对安全性的关注使我们能够快速安全地进行实验。"

— Dimitri Glazkov, Principal Engineer, Opal Team

了解更多： opal.google

Gemini Enterprise: Custom Agents for Business

Gemini Enterprise 使企业能够构建专门针对其特定工作流和数据的强大、自定义 AI 代理。

Gemini Enterprise 如何使用 A2UI：

正在集成 A2UI 以允许企业代理在业务应用程序中渲染 丰富、交互式的 UI——超越简单的文本响应，引导员工完成复杂的工作流程。

  • 数据输入表单: 用于结构化数据收集的 AI 生成表单
  • 审批仪表板: 用于审查和审批流程的动态 UI
  • 工作流自动化: 用于复杂任务的分步界面
  • 自定义企业 UI: 针对行业特定需求的定制界面

"我们的客户需要他们的代理不仅仅是回答问题；他们需要代理引导员工完成复杂的工作流程。A2UI 将允许在 Gemini Enterprise 上构建的开发人员让他们的代理生成任何任务所需的动态、自定义 UI，从数据输入表单到审批仪表板，从而极大地加速工作流自动化。"

— Fred Jabbour, Product Manager, Gemini Enterprise

了解更多： Gemini Enterprise

Flutter GenUI SDK: Generative UI for Mobile

Flutter GenUI SDK 将动态的、AI 生成的 UI 带到跨移动、桌面和 Web 的 Flutter 应用程序中。

GenUI 如何使用 A2UI：

GenUI SDK 使用 A2UI 作为底层协议，用于服务器端代理和 Flutter 应用程序之间的通信。当您使用 GenUI 时，您在底层使用的是 A2UI。

  • 跨平台支持: 同一个代理适用于 iOS, Android, web, desktop
  • 原生性能: Flutter widgets 在每个平台上原生渲染
  • 品牌一致性: UI 匹配您应用的设计系统
  • 服务器驱动的 UI: 代理控制显示内容，无需应用更新

"我们的开发人员选择 Flutter，因为它可以让他们快速创建富有表现力、品牌丰富、自定义的设计系统，在每个平台上都感觉很棒。A2UI 非常适合 Flutter 的 GenUI SDK，因为它确保每个平台上的每个用户都能获得高质量的原生感觉体验。"

— Vijay Menon, Engineering Director, Dart & Flutter

合作伙伴集成 (Partner Integrations)

AG UI / CopilotKit: Full-Stack Agentic Framework

AG UI 和 CopilotKit 为构建代理应用程序提供了一个全面的框架，具有 day-zero A2UI 兼容性。

它们如何协同工作：

AG UI 擅长在自定义前端与其专用代理之间创建高带宽连接。通过添加 A2UI 支持，开发人员可以获得两全其美的方案：

  • 状态同步: AG UI 处理应用状态和聊天历史
  • A2UI 渲染: 渲染来自第三方代理的动态 UI
  • 多代理支持: 协调来自多个代理的 UI
  • React 集成: 与 React 应用程序无缝集成

"AG UI 擅长在自定义构建的前端与其专用代理之间创建高带宽连接。通过添加对 A2UI 的支持，我们为开发人员提供了两全其美的方案。他们现在可以构建丰富的、状态同步的应用程序，这些应用程序还可以通过 A2UI 渲染来自第三方代理的动态 UI。这对于多代理世界来说是完美的匹配。"

— Atai Barkai, Founder of CopilotKit and AG UI

Google 的 AI 驱动产品

随着 Google 在全公司范围内采用 AI，A2UI 提供了一种 让 AI 代理交换用户界面的标准化方式，而不仅仅是文本。

内部代理采用：

  • 多代理工作流: 多个代理为同一个 surface 做贡献
  • 远程代理支持: 在不同服务上运行的代理可以提供 UI
  • 标准化: 跨团队的通用协议减少了集成开销
  • 外部暴露: 内部代理可以轻松地向外部暴露（例如，Gemini Enterprise）

"就像 A2A 让任何代理可以与另一个代理对话一样，无论平台如何，A2UI 标准化了用户界面层，并通过编排器支持远程代理用例。这对于内部团队来说非常强大，使他们能够快速开发代理，其中丰富的用户界面是常态，而不是例外。随着 Google 进一步进军生成式 UI，A2UI 提供了一个完美的平台，用于渲染在任何客户端上的服务器驱动的 UI。"

— James Wren, Senior Staff Engineer, AI Powered Google
